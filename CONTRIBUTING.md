# Contributing

Thank you for your interest in this project.

This repository is maintained as a maintainer-led project. External code changes to the repository are not accepted through pull requests.

## How to contribute

Use **[GitHub Issues](https://github.com/igorxut/rapidfuzz-collections/issues)** to:

- report reproducible bugs;
- propose concrete new features;
- report unclear or incorrect documentation.

Use **[GitHub Discussions](https://github.com/igorxut/rapidfuzz-collections/discussions)** for everything else:

- questions about usage;
- general feedback;
- ideas that are not yet concrete enough for an issue.

Please do not open pull requests. Pull requests may be closed without review.

## Bug reports

When reporting a bug, please include:

- package version;
- Python version;
- operating system;
- a minimal reproducible example;
- expected behavior;
- actual behavior, including traceback when relevant.

Issues without enough information may be closed if the problem cannot be reproduced.

## Feature requests

Feature requests are welcome, but they are treated as proposals.

Please describe:

- the problem you are trying to solve;
- the expected behavior;
- why the feature belongs in this library;
- alternatives or workarounds you considered.

Opening an issue does not guarantee that the feature will be implemented.

## Pull requests

Pull requests are not accepted.

All changes to this repository are made by the maintainer. If you want to propose a change, open an issue. The maintainer will decide whether and how to implement it.

## Documentation feedback

Documentation corrections and improvement suggestions should be reported through GitHub Issues.

## Licensing of feedback

By submitting an issue, suggestion, or other feedback, you agree that the maintainer may use the ideas and information you provide to improve the project under the project's license.

## Maintainer decisions

The maintainer has final discretion over:

- whether a proposal is accepted;
- how a feature is implemented;
- project scope and priorities.
